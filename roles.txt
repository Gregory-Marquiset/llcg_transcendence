Product Owner (PO): être la voix du projet et des utilisateurs
    Définir la vision : Décider ce que le projet doit accomplir et pourquoi
    Gérer le backlog (liste de priorités) : Maintenir la liste des fonctionnalités à développer, les prioriser
    Prendre les décisions : Trancher sur ce qu'on fait, ce qu'on reporte, ce qu'on abandonne
    Valider le travail : Vérifier que chaque fonctionnalité terminée correspond aux attentes
    Communiquer avec l'extérieur : Présenter le projet aux évaluateurs, répondre aux questions
En pratique => sait ce qu'on dev en priorité, tranche quand l'équipe hésite et s'assure qu'on est focus sur l'essentiel

Project Manager (PM): fait avancer le projet et resoud les problemes / facilitateur
    Organiser les rituels : Planning des sprints, daily meetings, rétrospectives
    Suivre l'avancement : Savoir où on en est, combien de temps il reste
    Fluidifier la communication : S'assurer que tout le monde est au courant de tout
    Débloquer les situations : Identifier et résoudre les obstacles (techniques, humains, organisationnels)
En pratique => time keeper du projet et des réunions anime les calls, suivi du board et des progressions et aide a débloquer les situations

Technical lead : garant qualité et cohérence techniques
    Définir l'architecture : Choisir comment structurer le code (frontend/backend, modules, etc.)
    Décider de la stack technique : Quel framework ? Quelle base de données ? Quels outils ?
    Assurer la qualité du code : Établir les normes, les conventions, les bonnes pratiques
    Reviewer le code critique : Valider les parties sensibles avant merge
En pratique => réponds aux questions "comment...?", lead les code review, empeche les dettes techniques et detecte a temps les pbs d'architecture 

Developer : code, teste, documente,...
    implémenter les features : Écrire le code des fonctionnalités assignées
    Participer aux code reviews : Relire et commenter le code des autres
    Tester son code : S'assurer que ça marche avant de push
    Documenter : Expliquer ce qui a été fait et comment ça fonctionne
En pratique => prend les taches du backlog (liste de taches en attente) et les developpe, creer des pull request, debug, aide et ecrit des tests dans la doc si nécéssaire
